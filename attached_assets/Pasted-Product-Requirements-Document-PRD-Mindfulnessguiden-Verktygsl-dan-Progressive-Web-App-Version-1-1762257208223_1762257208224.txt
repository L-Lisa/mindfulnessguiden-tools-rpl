Product Requirements Document (PRD)
Mindfulnessguiden Verktygslådan - Progressive Web App
Version: 1.0 Date: 2025-11-03  Deployment: Vercel or Netlify Target Platforms: iOS Safari, Chrome, DuckDuckGo (mobile & desktop)

1. PRODUCT OVERVIEW
1.1 Purpose
A lightweight Progressive Web App (PWA) providing mindfulness guides with a portable toolkit of exercises accessible on their phones. Functions as a lead magnet and practical resource for certified mindfulness facilitators.
1.2 Core Value Proposition
Simple, swipeable flashcard interface for quick access to mindfulness exercises
Works 100% offline once installed
No backend, no authentication, no data collection
Small footprint, fast loading
1.3 User Persona
Primary: Certified mindfulness guides from Mindfulnessguiden courses
Secondary: HR professionals, organizational developers interested in workplace mindfulness
Language: Swedish-speaking users in Sweden
Device: Primarily mobile (iPhone/Android), but must work on desktop

Vertically centered content

Clean, welcoming design

Use simple HTML/CSS for maximum accessibility

Consider detecting platform and showing relevant instructions first
3.1.3 Exercise Cards (Cards 1-10)
Content Structure:
Watermark: "mindfulnessguiden.se" (top right corner, subtle, small font)
Exercise Name: Large, bold heading
Duration: "Tid: X minuter" (for now, hardcoded random value between 4-15 minutes per card)
Instructions: Body text, scrollable if content exceeds card height
Logo: Small, centered horizontally at bottom of card (provided as TheLogo.png)
Navigation: Left/Right arrows visible on edges, center-aligned vertically
Interaction:
Swipe left/right to navigate (mobile)
Click arrows to navigate (desktop/mobile)
Smooth animated transitions between cards
Scroll within card if content is long (vertical scroll only within card)
3.2 Navigation System
3.2.1 Card Swiping
Touch gestures: Swipe left (next card) / Swipe right (previous card)
Mouse drag: Optional - allow click-and-drag on desktop
Keyboard: Arrow keys for desktop navigation
Animation: Smooth slide transition (300ms ease-out)
3.2.2 Navigation Arrows
Position: Fixed on left and right edges of viewport, vertically centered
Visibility: Always visible
Design:
Subtle, iOS-style chevron icons
Semi-transparent background circle/pill
Tap/click area: minimum 44x44px
Should not overlap content
Behavior:
Left arrow disabled on first card (cover card)
Right arrow disabled on last card (card 10)
Visual feedback on hover/press
3.2.3 Progress Indicator
Type: Simple dot indicator or "Card X of 11" text
Position: Bottom center or top center of card
Design: Minimal, not distracting
Behavior: Updates as user navigates

4. PROGRESSIVE WEB APP (PWA) REQUIREMENTS
4.1 Manifest File
File: /public/manifest.json
{
  "name": "Mindfulnessguiden Verktygslådan",
  "short_name": "Verktygslådan",
  "description": "Verktygslåda för certifierade mindfulnessguider",
  "start_url": "/",
  "display": "standalone",
  "background_color": "#F6F4EA",
  "theme_color": "#293556",
  "orientation": "portrait-primary",
  "icons": [
    {
      "src": "/icon-192.png",
      "sizes": "192x192",
      "type": "image/png",
      "purpose": "any maskable"
    },
    {
      "src": "/icon-512.png",
      "sizes": "512x512",
      "type": "image/png",
      "purpose": "any maskable"
    }
  ]
}
Icon Requirements:
Use provided TheLogo.png as base
Generate 192x192 and 512x512 PNG icons
Icons should have proper padding for iOS "maskable" requirement
Background should match brand colors
4.2 Service Worker
Requirements:
Cache all static assets (HTML, CSS, JS, images, JSON)
Implement "cache-first" strategy for maximum offline reliability
Update cache on new deployments
No network requests required after first load
Implementation:
// Use Vite PWA plugin with Workbox
// vite.config.js configuration:
{
  plugins: [
    VitePWA({
      registerType: 'autoUpdate',
      workbox: {
        globPatterns: ['**/*.{js,css,html,ico,png,svg,json}'],
        runtimeCaching: [
          {
            urlPattern: /^https:\/\/fonts\.googleapis\.com\/.*/i,
            handler: 'CacheFirst',
            options: {
              cacheName: 'google-fonts-cache',
              expiration: {
                maxEntries: 10,
                maxAgeSeconds: 60 * 60 * 24 * 365 // 1 year
              },
              cacheableResponse: {
                statuses: [0, 200]
              }
            }
          }
        ]
      },
      manifest: {
        // manifest.json content from above
      }
    })
  ]
}
4.3 Offline Functionality
100% offline operation after initial load
No error states for missing network
All assets bundled and cached
No external API calls
No analytics or tracking (fully privacy-respecting)

iOS HIG Principles:

Safe area insets respected (especially iOS notch/home indicator)
5.2 Typography Scale
/* Based on iOS type scale */
--font-size-small: 13px;      /* Captions, watermarks */
--font-size-body: 17px;        /* Body text, instructions */
--font-size-large: 22px;       /* Exercise duration */
--font-size-title: 28px;       /* Exercise names */
--font-size-display: 34px;     /* Cover card title */

--line-height-tight: 1.2;
--line-height-normal: 1.5;
--line-height-relaxed: 1.7;
5.3 Card Specifications
Dimensions:
Mobile: Full viewport width, appropriate height (min 70vh, max 100vh)
Desktop: Max width 600px, centered, appropriate height
Padding: 24px on all sides
Border radius: 16px (iOS-style rounded corners)
Shadow: Subtle elevation box-shadow: 0 4px 12px rgba(41, 53, 86, 0.08);

Scrolling Behavior:
Card container: Fixed height, no scroll
Content area: Scrollable vertically if content exceeds available space
Scroll indicators: Native browser scrollbar (minimal, auto-hiding on iOS)
5.4 Color Usage
:root {
  /* Brand colors */
  --color-primary: #293556;      /* Navy - headers, text */
  --color-secondary: #E6E6FA;    /* Lavender - accents */
  --color-background: #F6F4EA;   /* Cream - card background */
  
  /* Semantic colors */
  --color-text-primary: #293556;
  --color-text-secondary: rgba(41, 53, 86, 0.6);
  --color-border: rgba(41, 53, 86, 0.1);
  --color-overlay: rgba(41, 53, 86, 0.04);
}
5.5 Animation Standards
Timing:
Card transitions: 300ms ease-out
Button press: 150ms ease-in-out
Accordion open/close: 250ms ease-in-out
Transforms:
Use transform and opacity for animations (GPU-accelerated)
Avoid animating width, height, or left/right

6. COMPONENT STRUCTURE
6.1 Suggested Component Hierarchy
src/
├── components/
│   ├── CoverCard.jsx           # Landing card with intro and accordion
│   ├── ExerciseCard.jsx        # Individual exercise card
│   ├── CardContainer.jsx       # Wrapper handling swipe/navigation
│   ├── NavigationArrows.jsx    # Left/Right arrow buttons
│   ├── ProgressIndicator.jsx   # Dot indicator or counter
│   ├── Accordion.jsx           # Reusable accordion for install instructions
│   └── InstallInstructions.jsx # Content for installation steps
├── data/
│   └── exercises.json          # Exercise content
├── assets/
│   ├── TheLogo.png             # Provided logo
│   └── icons/                  # Arrow icons, etc.
├── styles/
│   ├── global.css              # CSS variables, resets
│   └── [component].module.css  # Component-specific styles
├── App.jsx                     # Main app component
└── main.jsx                    # Entry point
6.2 State Management
No external state library needed - use React's built-in hooks:
useState for current card index
useEffect for loading exercise data
useRef for touch gesture tracking
Context API only if needed for theme/settings (unlikely in v1)
6.3 Key Component Responsibilities
App.jsx
Load exercise data from JSON
Manage current card index
Handle keyboard navigation
Render CardContainer with exercises
CardContainer.jsx
Handle swipe gestures (touch events)
Animate card transitions
Manage card visibility
Render current card (Cover or Exercise)
ExerciseCard.jsx
Display exercise name, duration, instructions
Render logo at bottom
Handle scrollable content
Show watermark
CoverCard.jsx
Display welcome content
Render clickable mindfulnessguiden.se link
Include Accordion with install instructions
Logo display

19. RISK MITIGATION
19.1 Technical Risks
Risk: PWA installation issues on iOS
Mitigation: Clear instructions, test on multiple iOS versions
Risk: Swipe gestures conflict with browser navigation
Mitigation: Implement proper touch event handling, preventDefault where needed
Risk: Content too long for mobile screens
Mitigation: Scrollable content areas, test with various content lengths
19.2 User Experience Risks
Risk: Users don't understand how to install PWA
Mitigation: Prominent, clear instructions on cover card
Risk: Arrow buttons not discoverable
Mitigation: Always visible, clear visual design
